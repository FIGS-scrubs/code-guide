Shop Talk

Golden rule

For additions or contributions to this Code Guide, please open an issue on GitHub.

Every line of code should appear to be written by a single person, no matter the number of contributors.

Browser Support

Shopify

  • Firefox (latest)
  • Chrome (latest)
  • Safari (latest)
  • IE8+
  • Mobile Safari 6+

WordPress, Tumblr & Ghost

  • Firefox (latest)
  • Chrome (latest)
  • Safari (latest)
  • IE9+
  • Mobile Safari 6+

We do not "support" the following browsers:

  • Opera
  • Anything not listed above

Not supporting doesn't mean a broken website, it means a lack of testing.

Peer Review

We don't release code that hasn't been peer reviewed. Whether it's a two line change to some CSS or a complete refactoring of a JavaScript function, your pull request will always be reviewed by one of your peers.

Since all of your code is written on a different branch than master anyways, peer review is as simple as opening up a pull request and assigning it to somebody to review.

The Exception

The only exception to committing code that isn't up to the standards in this document is when working on an older theme. It's preferable to keep code consistent to the theme that you're working on even if that code isn't up to our current standards (within reason). It's important to document these decisions in your Pull Request.

If the theme still has shelf life (not going to be archived any time soon), create a new task detailing the changes that should be made to bring the theme up to our current standards.
